# Спецификация прецедентов

## UC-04: Создание бронирования

| Параметр | Описание |
|----------|----------|
| **ID** | UC-04 |
| **Название** | Создание бронирования |
| **Актор** | Пользователь, Администратор |
| **Предусловие** | Пользователь аутентифицирован |
| **Постусловие** | Создано новое бронирование |

### Основной поток

| Шаг | Действие пользователя | Реакция системы |
|-----|----------------------|-----------------|
| 1 | Выбирает комнату | Отображает календарь |
| 2 | Выбирает дату и время | Проверяет доступность |
| 3 | Вводит тему встречи | - |
| 4 | Нажимает "Забронировать" | Сохраняет бронирование |

### Альтернативный поток (комната занята)

| Шаг | Действие пользователя | Реакция системы |
|-----|----------------------|-----------------|
| 1 | Выбирает занятое время | Сообщение "Комната занята" |
| 2 | Выбирает другое время | Возврат к шагу 2 |

## UC-09: Управление пользователями

| Параметр | Описание |
|----------|----------|
| **ID** | UC-09 |
| **Название** | Управление пользователями |
| **Актор** | Администратор |
| **Предусловие** | Администратор аутентифицирован |
| **Постусловие** | Данные пользователя обновлены |

### Основной поток (создание)

| Шаг | Действие администратора | Реакция системы |
|-----|------------------------|-----------------|
| 1 | Переходит в "Пользователи" | Список пользователей |
| 2 | Нажимает "Добавить" | Форма создания |
| 3 | Вводит логин, пароль, ФИО | Проверка уникальности |
| 4 | Нажимает "Сохранить" | Сохраняет пользователя |
