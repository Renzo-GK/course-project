# ADR (Architecture Decision Records)

## ADR-001: Выбор архитектурного паттерна PCMEF

**Статус:** Принято

**Контекст:**
Необходимо выбрать архитектурный паттерн для веб-приложения.

**Решение:**
Использовать паттерн PCMEF (Presentation-Control-Mediator-Entity-Foundation).

**Обоснование:**
- Требование методички
- Четкое разделение ответственности
- Направленность зависимостей сверху вниз
- Возможность тестирования каждого слоя изолированно

**Последствия:**
- Строгая иерархия слоев
- Использование интерфейсов между слоями
- Сложность для небольших проектов, но оправдано для учебных целей

## ADR-002: Выбор технологического стека

**Статус:** Принято

**Контекст:**
Необходимо выбрать технологии для реализации веб-приложения.

**Решение:**

| Компонент | Технология |
|-----------|------------|
| Язык | Java 17 |
| Фреймворк | Spring Boot |
| Шаблонизация | Thymeleaf |
| ORM | Spring Data JPA |
| БД | PostgreSQL |
| Безопасность | Spring Security + BCrypt |
| Сборка | Maven |

**Обоснование:**
- Требования методички (траектория Б)
- Широкое распространение в индустрии
- Большое сообщество и документация
- Простота интеграции всех компонентов

## ADR-003: Использование Thymeleaf вместо React/Vue

**Статус:** Принято

**Контекст:**
Необходимо выбрать способ отображения веб-страниц.

**Решение:**
Использовать Thymeleaf (серверный рендеринг).

**Обоснование:**
- Простота изучения (знание HTML достаточно)
- Минимум JavaScript кода
- Безопасность (нет риска XSS через API)
- Соответствие учебным целям (оценка 3)

## ADR-004: Аутентификация через Spring Security

**Статус:** Принято

**Контекст:**
Необходимо обеспечить безопасность приложения.

**Решение:**
Использовать Spring Security с BCrypt для хеширования паролей.

**Обоснование:**
- Стандарт де-факто для Java-приложений
- Встроенная поддержка ролей (USER/ADMIN)
- BCrypt — безопасное хеширование
- Простая настройка

## ADR-005: Использование PostgreSQL

**Статус:** Принято

**Контекст:**
Необходимо выбрать базу данных.

**Решение:**
Использовать PostgreSQL.

**Обоснование:**
- Требование методички
- Open Source
- Поддержка всех необходимых функций
- Лёгкость использования с Hibernate
- Возможность заменить на H2 для разработки
