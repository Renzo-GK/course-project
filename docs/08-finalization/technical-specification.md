# Техническое задание

## 1. Общие сведения
- **Наименование системы:** Веб-сервис для бронирования переговорных комнат
- **Назначение:** Автоматизация бронирования переговорных комнат
- **Заказчик:** Учебный проект (курсовая работа)
- **Разработчик:** [Ваше ФИО]

## 2. Требования к системе

### 2.1. Функциональные требования
1. Аутентификация и авторизация пользователей
2. Просмотр списка переговорных комнат
3. Просмотр календаря занятости комнаты
4. Создание бронирования с проверкой доступности
5. Отмена бронирования
6. Управление пользователями (администратор)
7. Управление комнатами (администратор)
8. REST API для интеграции

### 2.2. Требования к архитектуре (PCMEF)
- Строгое разделение на слои: Presentation, Control, Mediator, Entity, Foundation
- Зависимости направлены сверху вниз
- Использование интерфейсов между слоями

### 2.3. Требования к базе данных
- PostgreSQL (разработка на H2)
- Нормализация до 3NF
- Индексы для оптимизации

### 2.4. Требования к интерфейсу
- Веб-интерфейс (Thymeleaf)
- 5+ страниц
- Адаптивная вёрстка

### 2.5. Требования к безопасности
- Spring Security
- BCrypt для хеширования паролей
- Разграничение ролей (USER/ADMIN)

## 3. Состав системы
1. Серверное приложение (Java + Spring Boot)
2. Веб-интерфейс (Thymeleaf + Bootstrap)
3. База данных (PostgreSQL)
4. REST API

## 4. Технологический стек
- Java 17
- Spring Boot 2.7.0
- Thymeleaf
- Spring Data JPA
- PostgreSQL / H2
- Spring Security
- Maven
- Git / GitHub
