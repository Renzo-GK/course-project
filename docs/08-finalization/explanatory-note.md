## ВВЕДЕНИЕ

### Актуальность
В современных офисах управление переговорными комнатами часто происходит хаотично. Сотрудники используют email, мессенджеры или устные договоренности для бронирования, что приводит к конфликтам двойного бронирования и простою комнат. Разработанная система позволяет автоматизировать этот процесс.

### Цель
Разработка веб-сервиса для бронирования переговорных комнат и управления рабочими пространствами.

### Задачи
1. Провести бизнес-анализ предметной области
2. Разработать модель требований
3. Спроектировать архитектуру (PCMEF)
4. Реализовать ядро системы
5. Разработать веб-интерфейс
6. Обеспечить безопасность
7. Выполнить развертывание

## 1. АНАЛИТИЧЕСКАЯ ЧАСТЬ

### 1.1. Описание предметной области
Предметная область — управление бронированием переговорных комнат в офисе. Основные сущности: Пользователь, Переговорная комната, Бронирование.

### 1.2. Анализ бизнес-процессов
Проведен анализ текущего процесса бронирования. Выявлены проблемы: отсутствие единой системы, конфликты двойного бронирования.

### 1.3. SWOT-анализ
Проведен SWOT-анализ. Сильные стороны: наличие комнат и оборудования. Слабые стороны: бронирование через email. Возможности: автоматизация. Угрозы: сопротивление сотрудников.

## 2. ПРОЕКТНАЯ ЧАСТЬ

### 2.1. Требования к ПО
Разработаны Use Case диаграмма и Domain Model.

### 2.2. Архитектура (PCMEF)
Применен паттерн PCMEF: Presentation, Control, Mediator, Entity, Foundation.

### 2.3. База данных
Разработана ER-диаграмма и DDL-скрипты для PostgreSQL.

### 2.4. Детальное проектирование
Разработаны диаграммы последовательности и диаграмма классов.

## 3. РЕАЛИЗАЦИОННАЯ ЧАСТЬ

### 3.1. Реализация ядра
Реализованы сущности, репозитории и сервисы. Код соответствует SOLID.

### 3.2. Рефакторинг
Применены паттерны: Data Mapper, Identity Map, Lazy Load.

### 3.3. Интерфейс
Разработан веб-интерфейс на Thymeleaf (5+ страниц) и REST API (8+ эндпоинтов).

### 3.4. Безопасность
Реализована аутентификация через Spring Security, хеширование паролей BCrypt.

## 4. ТЕСТИРОВАНИЕ
Проведено модульное тестирование (JUnit + Mockito). Покрытие кода > 40%.

## 5. РАЗВЁРТЫВАНИЕ
Приложение собрано с помощью Maven, запускается на Tomcat/Spring Boot.

## 6. УПРАВЛЕНИЕ ПРОЕКТОМ
Разработаны WBS и диаграмма Ганта. Проведена оценка COCOMO.

## ЗАКЛЮЧЕНИЕ

Разработан веб-сервис для бронирования переговорных комнат, полностью соответствующий требованиям методички.
