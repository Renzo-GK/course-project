## 2. БИЗНЕС-КОНТЕКСТ

### Проблема
В современных офисах управление переговорными комнатами происходит хаотично. Сотрудники используют email, мессенджеры или устные договоренности, что приводит к:
- Конфликтам двойного бронирования
- Простою комнат из-за отсутствия информации
- Трудностям с поиском свободной комнаты

### Решение
Создание веб-приложения, доступного через браузер, которое позволит:
- Просматривать расписание всех комнат в реальном времени
- Бронировать комнаты на нужное время
- Управлять списком комнат и пользователей

## 3. ЦЕЛИ ПРОЕКТА
1. Разработать веб-интерфейс (минимум 5 страниц)
2. Реализовать CRUD-операции для основных сущностей
3. Обеспечить аутентификацию и разграничение прав доступа
4. Создать REST API для интеграции
5. Разработать модульные тесты с покрытием > 40%
